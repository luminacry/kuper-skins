# 编辑器第 1 屏项目页规范

## 目标

首屏不再是图形参数页，而是项目页。

正确逻辑：

- 左侧语义先是 `项目夹`
- 当前项目里先看到 `组件`
- 进入组件以后，才看到 `方形 / 圆形 / 文本 / 图片` 等子项
- 右侧属性面板只在明确点击“调整”后出现，不能抢首页主结构

## 信息架构

统一层级：

1. 项目
2. 组件
3. 子项

子项可以是：

- 方形
- 圆形
- 文本
- 图片
- 更细的内部编组

## 首屏状态

默认进入时：

- 当前容器停在默认项目
- 底部激活页签固定为 `项目`
- 列表展示该项目下的组件
- 不自动跳入某个图形参数页
- 预览区允许没有高亮选中对象

进入组件时：

- 左侧条切换为 `组件`
- 底部列表展示该组件的直接子项
- 如果用户再点击具体图形，才进入真正的对象选中态

## 界面规则

顶部：

- 标题显示当前容器名称
- 项目层副标题为 `项目 · 组件列表`
- 组件层副标题为 `组件 · 子项列表`

左侧条：

- 项目层保留 Pencil 的 `项目夹` 素材
- 组件层显示 `组件`
- 组件层允许点击返回上一级项目

底部页签：

- 项目
- 层
- 位置
- 全局变量
- 动画

本轮只有 `项目` 作为真实首屏内容，其余页签保持占位。

## 交互规则

点击组件：

- 进入该组件
- 列表切换为组件子项
- 清空图形高亮，避免误导用户以为已经进入参数编辑

点击图形：

- 保持在当前组件列表
- 只更新选中状态与预览高亮
- 不自动切到底部参数页

点击右侧“调整”工具：

- 打开右侧属性面板
- 仅对具体图形或文本生效

## Android 对齐要求

- 演示文档本身必须也是 `项目 -> 组件 -> 子项` 树，不允许 UI 假装分层而底层仍是散图层
- 新增图形默认插入当前组件，不再直接挂到文档顶层
- 首页只保留项目列表逻辑，旧的 shape-first 完成态不再作为主路径
